# swagger-client
MUSTLink is a REST API primarly designed to offer easy access to parameter data from MUST and ARES archives. All endpoints are documented with detailed request/response information. Please refer to section \"3. Definitions\" for a list of all available models. Endpoints are also tagged and categorized according to their nature and/or behaviour, please refer to section \"1.3. Tags\" for a list of all available categories.  MUSTLink relies on a stateless authentication strategy where only authenticated users are able to access secured resources. It also offers additional authorization layers under the form of roles and groups. It is possible to restict the access to a given resource to be limited to a given group or role.  The user is required to authenticate using the user authentication endpoint and obtain a JWT (JSON Web Token) token. A user token is valid for a limited period of time (applicable through the system configuration) and should be included in every secured request header with \"Authorization\" key. This user-specific token identifies the user to MUSTLink and serves to check user access rights for the requested resources.  MUSTLink REST endpoints are designed to provide generic responses parsable using any JSON parser. WebMUST specific endpoints are provided for GUI elements like trees, tables, buttons or links, and are accessible under /web/ path.
